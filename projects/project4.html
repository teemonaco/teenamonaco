Research & Insights

User Personas

Our website's target demographic has been refined to women aged 55 and older. This age group often faces unique challenges when it comes to digital accessibility, particularly regarding vision.

Design Challenges

Common vision-related issues include presbyopia (difficulty focusing on close objects), cataracts (cloudy vision), and in some cases, more severe conditions like macular degeneration. To address these concerns, our design decisions must prioritize legibility, scalability, and accessibility, ensuring that users with various degrees of vision impairment can comfortably navigate and interact with our site.

Design Iterations

The design system went through several iterations to ensure a balance between time efficiency and a consistent, accessible user experience across all devices.

Initial Component Library

The first version of the library focused on essential components, such as buttons, input fields, and layout grids. User testing revealed opportunities to further streamline these for better reusability.

Refining for Consistency

In later iterations, we focused on making the components more scalable and flexible, while improving their consistency across different devices and platforms.

Prototypes

High-fidelity prototypes were developed in Figma to simulate how the component library and dashboard would function across various scenarios.

Master Component Library

This prototype showcased the full set of reusable components, designed to enhance time efficiency by enabling easy copy-paste for development teams.

WCAG AA Compliance

The design system adhered to accessibility standards, ensuring color contrast, text sizes, and interactive elements were compliant with WCAG AA.

Final Product: Internal Design System Dashboard

The final product is a tailored internal design system dashboard, providing developers with a streamlined copy-paste code library for consistent implementation, adhering to accessibility standards.

Dashboard for Reusable Code

Developers can access reusable code directly from the dashboard, ensuring consistency in design and reducing development time.

WCAG AA Compliance for All Components

All components in the design system passed WCAG AA standards, ensuring that the interface is accessible for users with visual impairments.
